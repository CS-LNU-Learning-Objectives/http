CERN httpd

Web server session

In order to associate a request to any other request, you need a way to store user data between HTTP requests.

  • The user send the state information with every request
    • http://www.example.com?id=1023&username=thajo
    • Cookie with all the data
  • Let the user send a "secret" id and let the server store the session data
    • http://www.example.com?id=sdhjH223Hsjhdj2jasdlj
    • Session cookie - encrypted or a hash
    • A by the server signed token (JSON Web Token)
